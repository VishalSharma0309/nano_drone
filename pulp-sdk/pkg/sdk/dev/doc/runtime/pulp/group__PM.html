◆ rt_pm_state_flags_e

enum rt_pm_state_flags_e

Describes the way to change power states.

Enumerator
RT_PM_STATE_FAST 

Fast mode. The runtime does the minimum to change the power state. When going to sleep mode, only retentive data are kept and the execution will start again from the main after wake-up. The same when going to deep mode but no data is kept. In both cases, every resource must be reallocated or reopened (e.g. memory allocation, cluster or camera).

Describes the power state before the wakeup.

Enumerator

Cold boot. The chip was off before waking up.

RT_PM_WAKEUP_SLEEP 

Sleep boot. The chip was insleep mode before waking up.

◆ rt_pm_wakeup_gpio_mode_e

enum rt_pm_wakeup_gpio_mode_e

Describes the way a GPIO can wakeup the chip.

Enumerator
RT_PM_WAKEUP_GPIO_RISING_EDGE 

Rising edge. The GPIO can wakeup the chip when switching from 0 to 1.

RT_PM_WAKEUP_GPIO_FALLING_EDGE 

Falling edge. The GPIO can wakeup the chip when switching from 1 to 0.

RT_PM_WAKEUP_GPIO_HIGH 

Level high. The GPIO can wakeup the chip when it is 1.

RT_PM_WAKEUP_GPIO_LOW 

Level low. The GPIO can wakeup the chip when it is 0.

◆ rt_pm_wakeup_clear_all()

void rt_pm_wakeup_clear_all ( )

This clears all the pending wakeup signals so that the chip can go to deep sleep mode. Any wakeup source (like RTC or GPIO) occuring after this call will be buffered again and will prevent the chip from going to deep sleep.
