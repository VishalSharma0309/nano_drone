Description

The I2C driver provides support for transferring data between an external I2C device and the chip running this driver.

◆ rt_i2c_close()

void rt_i2c_close ( rt_i2c_t *  handle,
rt_event_t *  event 
)

This function can be called to close an opened I2C device once it is not needed anymore, in order to free all allocated resources. Once this function is called, the device is not accessible anymore and must be opened again before being used. This operation is asynchronous and its termination can be managed through an event.

Parameters
handleThe handler of the device which was returned when the device was opened.
eventThe event used for managing termination.

◆ rt_i2c_open()

rt_i2c_t* rt_i2c_open ( char *  dev_name,
rt_i2c_conf_t *  conf,
rt_event_t *  event 
)

This function must be called before the I2C device can be used. It will do all the needed configuration to make it usable and also return a handle used to refer to this opened device when calling other functions. This operation is asynchronous and its termination can be managed through an event.

Parameters
dev_nameThe device name in case the connection must be configured automatically. This name should correspond to the one used to configure the devices managed by the runtime. If it is NULL, all connection information must be provided in the configuration structure (interface id and chip select).
confA pointer to the I2C configuration. Can be NULL to take default configuration.
eventThe event used for managing termination.
Returns
NULL if the device is not found, or a handle identifying the device and which can be used with other functions.

◆ rt_i2c_read()

void rt_i2c_read ( rt_i2c_t *  handle,
unsigned char *  rx_buff,
int  length,
int  xfer_pending,
rt_event_t *  event 
)

This function can be used to read at least 1 byte of data from the I2C device. The copy will make an asynchronous transfer between the I2C and one of the chip memory. An event can be specified in order to be notified when the transfer is finished.

Parameters
handleThe handler of the device which was returned when the device was opened.
rx_buffThe address in the chip where the received data must be written.
lengthThe size in bytes of the copy
xfer_pendingIf 1, the stop bit is not generated so that the same transfer can be continued with another call to rt_i2c_read or rt_i2c_write
eventThe event used for managing termination.

int  length,
int  xfer_pending,
rt_event_t *  event 
)

This function can be used to write at least 1 byte of data to the I2C device. The copy will make an asynchronous transfer between the I2C and one of the chip memory. Compared to rt_i2c_write, the START bit is not generated and the slave address is not sent. An event can be specified in order to be notified when the transfer is finished.

Parameters
handleThe handler of the device which was returned when the device was opened.
tx_dataThe address in the chip where the data to be sent.
lengthThe size in bytes of the copy
xfer_pendingIf 1, the stop bit is not generated so that the same transfer can be continued with another call to rt_i2c_read or rt_i2c_write
eventThe event used for managing termination.
