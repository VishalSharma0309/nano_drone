Description

This structure is used to pass the desired task scheduler configuration to the runtime when it is opened.

Field Documentation

◆ cid

int rt_task_conf_t::cid

Cluster ID where the task scheduler must be created.

◆ nb_cores

int rt_task_conf_t::nb_cores

Number of cluster cores which will pop and execute tasks. If it is zero, this will use the maximum number of cores.

◆ stack_size

unsigned int rt_task_conf_t::stack_size

Stack size to be used by the cluster cores when executing the tasks. The size is the size of the stack for one core. If the size is 0, no stack is given to the cores, which means the stack will have to be provided by the tasks.

◆ stacks

void* rt_task_conf_t::stacks

Stacks to be used by the cluster cores when executing the tasks. If it is NULL, and the stack size is not 0, the runtime will allocate default stacks. If it is not NULL, this should point to the lowest address of the area containing the stacks for all cores.
