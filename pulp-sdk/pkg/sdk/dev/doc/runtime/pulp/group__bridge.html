Description

This API provides support for interacting with an external bridge for example to access files on the workstation.

Enumeration Type Documentation

◆ rt_fb_format_e

enum rt_fb_format_e

This is used to specify what is the format of the image sent by the chip to the external bridge when updating a framebuffer.

Enumerator
RT_FB_FORMAT_GRAY 

Each pixel is gray on 8 bits.

RT_FB_FORMAT_RGB 

Each pixel is 3 x8bits channels (RGB).

RT_FB_FORMAT_RAW 

Each pixel is 3 x8bits channels (RGB).

◆ rt_bridge_close()

int rt_bridge_close ( int  file,
rt_event_t *  event 
)

This can be called to ask the debug bridge to close a file on the workstation. Can only be called from fabric controller. This operation is asynchronous and its termination can be managed through an event.

Parameters
fileThe file descriptor returned when the file was opened.
eventThe event used for managing termination. This event can only be NULL or a blocking event (the callback mode is not possible).
Returns
If the event parameter is NULL, this returns the return value of the call to close, otherwise it is undefined.

◆ rt_bridge_connect()

int rt_bridge_connect ( int  wait_bridge,
rt_event_t *  event 
)

This must be called before calling any other function of this API in order to connect the runtime to the external bridge. The specified event will terminate only when the runtime has established the connection with the external bridge. Can only be called from fabric controller. This operation is asynchronous and its termination can be managed through an event.

Parameters
wait_bridgeIf 1, this will block the caller until the bridge is connected.
eventThe event used for managing termination. This event can only be NULL or a blocking event (the callback mode is not possible).
Returns
1 if the connection was successfull, 0 otherwise.

◆ rt_bridge_disconnect()

void rt_bridge_disconnect ( rt_event_t *  event)

This can be called close down the connection with the external bridge. No other function from this API can be called after this one is called unless the connection is opened again. Can only be called from fabric controller. This operation is asynchronous and its termination can be managed through an event.

Parameters
eventThe event used for managing termination.

This can be called to ask the debug bridge to open a framebuffer on the workstation. Can only be called from fabric controller. This operation is asynchronous and its termination can be managed through an event.

Parameters
nameThe name associated to this framebuffer, which is passed to the window manager so that the window of this framebuffer can be easily identified.
widthThe width in pixels of the framebuffer.
heightThe height in pixels of the framebuffer.
formatThe format of the image sent by the chip to the bridge to update the framebuffer.
eventThe event used for managing termination. This event can only be NULL or a blocking event (the callback mode is not possible).
Returns
If the event parameter is NULL, this returns the framebuffer descriptor which can be used when updating the framebuffer, otherwise it is undefined.

◆ rt_bridge_fb_update()

void rt_bridge_fb_update ( uint64_t  fb,
unsigned int  addr,
int  posx,
int  posy,
int  width,
int  height,
rt_event_t *  event 
)

This can be called to ask the debug bridge to update a framebuffer on the workstation. It is possible to update either the whole framebuffer or just a window. Can only be called from fabric controller. This operation is asynchronous and its termination can be managed through an event.

Parameters
fbThe framebuffer descriptor which was returned when the framebuffer was opened.
addrThe address of the buffer to use to update the framebuffer.
posxThe first column of the window in the framebuffer where pixels must be updated. Can be -1 to update the whole framebuffer.
posyThe first row of the window in the framebuffer where pixels must be updated. Ignored when updating the whole framebuffer.
widthThe width in pixels of the window to be updated in the framebuffer. Ignored when updating the whole framebuffer.
heightThe height in pixels of the window to be updated in the framebuffer. Ignored when updating the whole framebuffer.
eventThe event used for managing termination. This event can only be NULL or a blocking event (the callback mode is not possible).
