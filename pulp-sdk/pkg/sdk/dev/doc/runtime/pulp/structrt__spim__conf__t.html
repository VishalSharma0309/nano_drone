Description

This structure is used to pass the desired SPI master configuration to the runtime when opening a device.

Field Documentation

◆ big_endian

char rt_spim_conf_t::big_endian

If 1, the elements are stored in memory in a big-endian way, i.e. the most significant byte is stored at the lowest address. This is taken into account only if the wordsize is 32 bits.

◆ cs

signed char rt_spim_conf_t::cs

If cs_gpio is -1, the normal chip select pins are used and this field specifies which one to use for the device.

◆ cs_gpio

signed char rt_spim_conf_t::cs_gpio

If it is different from -1, the specified number is used to drive a GPIO which is used as a chip select for the SPI device. The cs field is then ignored.

◆ id

signed char rt_spim_conf_t::id

If it is different from -1, this specifies on which SPI interface the device is connected.

◆ max_baudrate

int rt_spim_conf_t::max_baudrate

Maximum baudrate for the SPI bitstream which can be used with the opened device .

◆ phase

char rt_spim_conf_t::phase

Phase of the clock.

◆ polarity

char rt_spim_conf_t::polarity

Polarity of the clock.

◆ wordsize

char rt_spim_conf_t::wordsize

Wordsize of the elements in the bitstream. Can be RT_SPIM_WORDSIZE_8 for 8 bits data or RT_SPIM_WORDSIZE_32 for 32 bits data. This is used to interpret the endianness.
