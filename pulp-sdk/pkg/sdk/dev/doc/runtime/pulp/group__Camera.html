Description

This API provides support for capturing images from an image sensor. It also provides an API for configuring and controlling an image sensor.

*argA pointer to the arguments of the command.

◆ rt_camera_capture()

static void rt_camera_capture ( rt_camera_t *  handle,
void *  buffer,
size_t  size,
rt_event_t *  event 
)
inlinestatic

Queue a buffer that will receive Camera samples. It is possible to call this function asynchronously and several times in order to queue several buffers. At a minimum 2 buffers should be queued to ensure that no data sampled is lost. This is also the most efficient way to retrieve data from the Camera device. You should always make sure that at least 2 buffers are always queued, by queuing a new one as soon as the current one is full. Can only be called from fabric-controller side.

Parameters
handleThe handle of the device which was returned when the device was opened.
bufferThe memory buffer where the captured samples will be transfered.
sizeThe size in bytes of the memory buffer.
eventThe event used for managing termination. The event will be notified as soon as the specified amount of bytes have been captured.

◆ rt_camera_close()

static void rt_camera_close ( rt_camera_t *  handle,
rt_event_t *  event 
)
inlinestatic

This function can be called to close an opened Camera device once it is not needed anymore in order to free all allocated resources. Once this function is called, the device is not accessible anymore and must be opened again before being used. This operation is asynchronous and its termination can be managed through an event. Can only be called from fabric-controller side.

Parameters
handleThe handle of the device which was returned when it was opened.
eventThe event used for managing termination.

◆ rt_camera_open()

rt_camera_t* rt_camera_open ( char *  dev_name,
rt_cam_conf_t *  conf,
rt_event_t *  event 
)

This function must be called before the Camera device can be used. It configure the device and return a handle that can be used to refer to the opened device when calling other functions. This operation is asynchronous and its termination can be managed through an event. Can only be called from fabric-controller side.

Parameters
dev_nameThe device name. This name should correspond to the one used to configure the devices managed by the runtime.
confA pointer to the Camera device configuration. Can be NULL to use the default configuration.
eventThe event used for managing termination.
Returns
NULL if the device is not found, or a handle identifying the device which can be used with other Camera functions.
